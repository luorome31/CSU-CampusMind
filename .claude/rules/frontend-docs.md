# Frontend Documentation Rules

## Required Reading at Task Start

Before starting any frontend development task, read these three files:

| File | Purpose |
|------|---------|
| `frontend/docs/frontend-progress-log.md` | Tracks ongoing and completed development work |
| `frontend/docs/frontend-question-log.md` | Records encountered issues and solutions |
| `frontend/docs/ARCHITECTURE.md` | Documents frontend architecture decisions and design |

## Required Update at Task End

After completing a frontend task, update the relevant documentation:

1. **`frontend-progress-log.md`** - Record the completed task, date, and key changes
2. **`frontend-question-log.md`** - If issues were encountered and resolved, document them
3. **`ARCHITECTURE.md`** - If architectural decisions were made, update accordingly

## Update Log Format

When updating progress log, add an entry:

```markdown
| <date> | <version> | <completed task> | <notes> |
```

When updating question log for new issues, follow the existing format in the file.
